# AI Smart Notebook – AI Mentor (Core AI Experience)

The AI Mentor is the heart of the AI Smart Notebook ecosystem. It is much more than a chatbot—it is an intelligent educational assistant that understands the user, the curriculum, and the learning journey. Design and implement this module as a premium, production-ready feature while maintaining the existing design language, branding, colors, typography, and UI consistency.

---

# 1. AI Mentor Entry Point

The AI Mentor is accessible from the header and must be available throughout the entire application for all user roles:

* Student
* Teacher
* Administrator

Clicking the AI Mentor button should never navigate to a new page.

Instead, it should open as a resizable side workspace.

---

# 2. Workspace Layout

When the AI Mentor is opened:

* It should appear as a side panel within the current screen.
* The main content should remain visible.
* The panel should open from the right side by default.
* The panel should support smooth opening and closing animations.

The layout should resemble a professional productivity workspace rather than a popup.

Example:

---

| Current Screen |           |
| -------------- | --------- |
|                |           |
|                | AI Mentor |
|                |           |
|                |           |

---

---

# 3. Adjustable Width

The AI Mentor panel must be fully resizable.

Users should be able to drag the divider to increase or decrease the panel width.

Requirements:

* Smooth dragging
* Minimum width
* Maximum width
* Responsive resizing
* Remember the user's preferred width during the session

The resizing behavior should match the notebook/PDF split-screen implementation to maintain consistency across the application.

---

# 4. Universal AI Interface

The same AI Mentor interface should be used across all roles:

* Student
* Teacher
* Administrator

Maintain a consistent layout, interactions, animations, and visual design.

Only the available tools, permissions, and AI capabilities should change depending on the user's role.

---

# 5. Modern AI Chat Interface

The AI Mentor should provide a polished conversational interface inspired by modern AI assistants while remaining unique to the AI Smart Notebook ecosystem.

Include:

* Conversation history
* User messages
* AI responses
* Typing animation
* Streaming responses
* Suggested prompts
* Rich message cards
* Markdown support
* Code blocks (when applicable)
* Mathematical expressions
* Tables
* Images (future support)
* File references (future support)

Design the interface to feel clean, premium, and distraction-free.

---

# 6. Personalized AI Knowledge

The AI Mentor should not behave like a generic chatbot.

It should be aware of the student's educational environment and provide personalized assistance using approved institutional data.

Examples of available context include:

For Students:

* Subjects
* Chapters
* Current syllabus
* Homework
* Assignments
* Notes
* Test history
* Progress
* Timetable
* Attendance
* Learning goals

For Teachers:

* Classes
* Student lists
* Lesson plans
* Assignments
* Tests
* Performance reports

For Administrators:

* School information
* Staff
* Notices
* Reports
* Academic management

The AI should use only authorized information relevant to the logged-in user.

---

# 7. AI Tutor Mode

One of the most important features is the dedicated AI Tutor Mode.

Within the AI Mentor interface, provide a clearly visible mode selector that allows users to switch between:

* AI Assistant
* AI Tutor Mode

The default mode should remain the general AI Assistant.

---

# 8. AI Tutor Mode Experience

When AI Tutor Mode is activated, the interaction should change completely.

The AI should behave like a real personal tutor rather than a standard chatbot.

Its purpose is to teach concepts step by step, encourage understanding, and adapt to the student's pace.

---

# 9. Learning Flow

When Tutor Mode starts:

Ask the student what they want to study.

Example:

"What would you like to learn today?"

Display available subjects as interactive cards or buttons.

Examples:

* Mathematics
* Science
* English
* History
* Geography

After selecting a subject, display available chapters.

Once a chapter is selected, begin the lesson.

---

# 10. Teaching Style

The AI Tutor should:

* Explain concepts gradually.
* Break complex ideas into smaller sections.
* Use simple language first, then introduce advanced details when appropriate.
* Provide real-life examples.
* Ask questions throughout the lesson.
* Encourage curiosity and active participation.
* Continuously check whether the student understands before moving on.

The experience should closely resemble learning with a dedicated personal tutor.

---

# 11. Interactive Teaching

Throughout the lesson, the AI Tutor should:

* Ask follow-up questions.
* Encourage the student to answer.
* Offer hints instead of immediately revealing answers.
* Correct misconceptions politely.
* Reinforce key concepts.
* Adjust explanations based on the student's responses.

The interaction should be conversational, adaptive, and supportive.

---

# 12. End of Lesson

At the end of each lesson, the AI Tutor should automatically provide:

* A concise summary of the topic.
* Key takeaways.
* Important formulas or definitions (when applicable).
* Common mistakes to avoid.
* Suggested revision points.
* Practice questions.
* A recommendation for the next topic.
